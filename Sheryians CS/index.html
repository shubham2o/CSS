% v/s vw or vh

• % takes its value from its parent every-time.

• On the other hand, vw or vh takes its value from the output screen.




em v/s rem

• They both are relative units, meaning their size is determined based on another value, typically the font size of an element.

• em unit is relative to the font size of its parent element. For example, if the font size of a parent element is 16px, 1em is equivalent to 16px. If you set font-size: 2em; on a child element, it would be twice the size of its parent's font size.

• rem unit is relative to the root element's font size, which is usually the <html> element. So, if the root element's font size is set to 16px, 1rem is equivalent to 16px regardless of the font size of its parent elements.

Block elements v/s inline elements v/s inline-block elements

• Block elements are elements that always start on a new line and take up the full width available to them. For example, if you use a <div> or <p> tag, it will create a new section that takes up the full width available.

• Inline elements flow with the text, meaning they don't start on a new line. They only take up as much space as necessary. For example, <span> or <a> tags.

• Inline-Block elements are a bit interesting. They take up horizontal space like block elements, but they don't start on a new line. It means they take up the width but don't force a line break. An example is the <imp> tag.

• So, if you are converting an image from inline-block to block, it means you want that image to start on a new line and take up the full width.

CSS Background



Flexbox

• It allows you to align elements horizontally and vertically, distribute space between them, make them responsive. You set up a flex container, which is like a box to hold your elements, and then you place flex items inside it.

• X axis = Horizontal (-) = MAIN = justify-content

• Y axis = Vertical (|) = CROSS = align-items



pseudo-elements (::)

Pseudo-elements in CSS are imaginary elements that allow you to style specific parts of an element, like the first letter, first-line, or last-child

They are indicated by " :: " in CSS

They are handy when you want to achieve specific design effects that can't be done with regular elements alone


pseudo-classes (:)

• Pseudo-classes in CSS are like special tags that help you style elements on their current state or interaction.

• They are indicated by " : " in CSS.

• For example, you might want to change the color of a button when someone hovers over it, or you want to change the color of a link when someone click on it.


Grid

• Grid in CSS is a way to create a structure for laying out web content. It allows you to divide a webpage into rows and columns, similar to how a table works.

• This gives you flexibility to arrange different elements of your webpage, like text and images, in an organized manner.

• It makes your webpage responsive, meaning it will look good on different screen sizes. Using a grid helps you have more control over the design of your webpage and makes it easier to keep everything neat and tidy.


Grid v/s Flexbox

• Grid allows you to create a grid-like structure with rows and columns, giving you precise control over the layout of your elements. It is multi-dimensional which means it can work on both X-axis (rows) and Y-axis (columns).

• Flexbox, on the other hand, lets you arrange elements in a single line, like lining up books on a shelf. It's more flexible when you need to arrange items in a single row or column. It is single dimensional which means it can work only on one, either on X-axis (rows) or on Y-axis (columns).

• So, if you need a precise grid layout with rows and columns, Grid might be a better choice. But if you need to arrange items in a single line, Flexbox is more suitable. Sometimes, you can even use both depending on your design needs.
